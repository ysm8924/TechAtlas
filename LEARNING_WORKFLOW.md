# TechAtlas 学习与开发流规范 📖

本文档旨在规定 **TechAtlas** 项目中的日常学习模式、代码存放规范以及与 AI 协作的最佳实践。

## 1. 核心学习模式：“由浅入深，原理与实战结合”

日常的技术探索（如协程、Flow、Compose、架构）将遵循以下四大步骤：
1. **基础认知与案例运行**：通过极简的 Demo 代码了解 API 的作用，并直观看到运行结果。
2. **源码与底层原理剖析**：结合 Kotlin Bytecode 反编译、源码断点或注释，深入了解其背后的原理（如状态机、CPS转换）。
3. **高频考点/易错点总结**：把常见的面试题、开发中容易踩坑的场景（如内存泄漏、协程取消失效）写成测试用例。
4. **业务实战模拟**：将学到的知识整合，模拟真实的业务场景（如：倒计时、并发网络请求重试）。

## 2. 应该在哪里添加代码/文档？(代码放置路由规则)

面对一个新的学习需求或功能模块，请按照以下“决策树”判断代码存放在何处：

### 🌳 Q1: 这是一个纯语言特性 / 算法，还是需要 Android SDK？
*   **纯语言特性 (如：协程原理、Kotlin 语法糖、反射、设计模式)**
    *   👉 **归宿**：`:labs:kotlin_core` 或 `:labs:java_core`。
    *   **原因**：这些模块不依赖 Android，有一个单纯的 `main()` 函数即可执行，编译仅需 1 秒钟，效率极高。
*   **依赖 Android SDK 或第三方 Android 库 (如：ViewModel, Room, 各种 Context)**
    *   👉 **归宿**：`:app` 或 `:core:xxx`，进入下一个问题。

### 🌳 Q2: 这是一个 UI 界面探索，还是底层基础封装？
*   **UI 布局 / Compose 特性 / 页面导航**
    *   👉 **归宿**：通用组件放在 `:core:ui`；完整的页面和交互放在 `:app`。
*   **网络框架 (Retrofit) / 数据库 (Room) / 基础业务逻辑**
    *   👉 **归宿**：分别放入对应的 `:core:network` / `:core:database` 等业务无关的底层库。

## 3. 注释与文档编写规范

1. **Lab 运行入口 (`xxxLab.kt`)**：
   每个知识点都应封装在一个特定的 `Object` 中（例如 `CoroutinesBasicLab`），暴露出 `runTests()` 方法，统一在各模块的 `LabRunner.kt` 或 `MainActivity.kt` 中调用。
2. **代码级注释**：
   示例代码必须伴随丰富的注释。注释需包含：**“概念解释”**、**“使用场景”**、**“底层原理(可选)”**。
3. **沉淀文档**：
   当一个大章节（例如：协程全系列）学习完毕时，可将散落的代码知识点提炼总结，在项目根目录或 `docs/` 目录下生成 Markdown 总结文档，以便日后复习。

## 4. 日常学习沟通方式 (AI 协助)

当你需要开启一个新的学习主题时，推荐使用以下提示词模板：
> *"我准备开始学习 [某知识点，如 Channel]。请在 [指定模块，如 kotlin_core] 帮我搭建好第一节课的 Lab 文件，给出由浅入深的步骤，包含必要的代码示例、注释和运行入口。"*
