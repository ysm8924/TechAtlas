# TechAtlas 项目工程结构图

这是一个基于 Google 现代应用架构 (MAD) 和 Clean Architecture 设计的 Android 项目结构。

项目旨在作为“技术展示容器”，包含业务功能模块 (Features) 和 技术实验模块 (Labs)。

## 📁 根目录 (Project Root)

```
TechAtlas/
├── build-logic/                  # 【面试亮点】自定义复合构建 (Composed Builds)
│   ├── convention/               # 统一管理 Gradle 插件和构建逻辑
│   │   ├── src/main/kotlin/
│   │   │   ├── AndroidApplicationConventionPlugin.kt
│   │   │   ├── AndroidLibraryConventionPlugin.kt
│   │   │   ├── AndroidFeatureConventionPlugin.kt  # 统一 Feature 模块配置
│   │   │   └── AndroidHiltConventionPlugin.kt
│   │   └── build.gradle.kts
│   └── build.gradle.kts
│
├── gradle/
│   └── libs.versions.toml        # Version Catalog：统一管理依赖版本
│
├── app/                          # [壳工程] 应用入口与容器
├── core/                         # [基础层] 通用能力下沉
├── features/                     # [业务层] 完整业务，遵循 Clean Architecture
├── labs/                         # [实验层] 技术深度探索与验证 (Java/Algo/Perf)
│
├── settings.gradle.kts           # 模块包含配置
└── build.gradle.kts              # 根构建脚本
```

## 📱 1. 壳工程 (:app)

**定位**：负责将积木拼装在一起，初始化全局环境，不包含具体业务逻辑。

```
app/
├── build.gradle.kts              # 依赖所有 features 和 labs
├── src/main/AndroidManifest.xml
└── src/main/java/com/TechAtlas/app/
    ├── TechAtlasApplication.kt      # @HiltAndroidApp，全局初始化
    ├── MainActivity.kt           # 单 Activity 架构入口
    ├── di/                       # 依赖注入聚合
    │   └── FeatureModule.kt      # 收集所有 IFeatureDemo 的实现 (插件化注册)
    └── navigation/
        └── AppNavHost.kt         # 顶层路由导航 (Compose Navigation)
```

## 🛠️ 2. 基础能力层 (:core)

**定位**：支撑上层业务的公共组件，模块间高度解耦。

```
core/
├── common/                       # 通用基础
│   └── src/main/java/com/TechAtlas/core/common/
│       ├── base/                 # BaseViewModel, BaseActivity
│       ├── dispatcher/           # CoroutineDispatchers (方便单元测试注入)
│       ├── result/               # Result<T> 封装
│       └── provider/             # 【核心】模块间通信接口
│           └── IFeatureDemo.kt   # 定义"插拔式"实验模块的标准接口
│
├── network/                      # 网络基础
│   └── src/main/java/com/TechAtlas/core/network/
│       ├── di/                   # Hilt Module (提供 Retrofit/OkHttp 单例)
│       ├── interceptor/          # Token 拦截器, 日志拦截器
│       └── model/                # NetworkResponse (通用泛型响应)
│
├── database/                     # 数据库基础
│   └── src/main/java/com/TechAtlas/core/database/
│       ├── di/                   # Hilt Module
│       ├── dao/                  # 通用 DAO 接口
│       └── AppDatabase.kt        # Room Database 定义
│
└── ui/                           # 设计系统 (Design System)
    └── src/main/java/com/TechAtlas/core/ui/
        ├── theme/                # Compose Theme (Color, Type, Shape)
        ├── icons/                # App 图标集
        └── components/           # 通用 UI 组件 (LoadingView, EmptyView)
```

## 🚀 3. 业务功能层 (:features)

定位：面试的核心展示区。严格遵循 Clean Architecture 分层原则。

示例模块：:features:news (新闻/博客阅读器)

```
features/
└── news/
    ├── build.gradle.kts
    └── src/main/java/com/TechAtlas/feature/news/
        │
        ├── domain/                     # 【内层 - 纯 Java/Kotlin】业务核心
        │   ├── model/                  # 业务实体 (解耦，不依赖 JSON 结构)
        │   │   └── NewsArticle.kt      
        │   ├── repository/             # 仓库接口定义 (依赖倒置)
        │   │   └── NewsRepository.kt   
        │   └── usecase/                # 业务逻辑单元 (面试重点)
        │       └── GetTopNewsUseCase.kt
        │
        ├── data/                       # 【外层 - Android/Lib实现】
        │   ├── di/                     # Hilt Module (Binds 接口到实现)
        │   ├── repository/             # 仓库实现
        │   │   └── NewsRepositoryImpl.kt  # 协调 Local/Remote 数据源策略
        │   ├── source/
        │   │   ├── local/              # Room 数据库相关
        │   │   │   ├── NewsDao.kt
        │   │   │   └── entity/NewsEntity.kt
        │   │   └── remote/             # Retrofit 网络相关
        │   │       ├── NewsApi.kt
        │   │       └── dto/NewsDto.kt  # Data Transfer Object
        │   └── mapper/                 # 数据清洗 (DTO -> Domain -> Entity)
        │       └── NewsMapper.kt       
        │
        └── presentation/               # 【UI 层 - MVVM / MVI】
            ├── NewsViewModel.kt        # 持有 UseCase
            ├── mvi/                    # MVI 架构实践
            │   ├── NewsIntent.kt       # 用户意图 (Refresh, Click)
            │   └── NewsUiState.kt      # UI 状态 (LCE: Loading/Content/Error)
            └── ui/
                ├── NewsScreen.kt       # Compose 界面
                └── components/         # 模块私有组件
```

## 🧪 4. 实验模块层 (:labs)

定位：深度技术练兵场，"脏"代码隔离区。侧重于单一技术点的底层验证。

特点：结构扁平，注重原理演示。

### ☕ A. Java 核心实验室 (:labs:java_core)

*考察点：反射、泛型、动态代理、注解*

```
labs/
└── java_core/
    ├── src/main/java/com/TechAtlas/lab/javacore/
    │   ├── FeatureImpl.kt           # 实现 IFeatureDemo，自动注册到首页
    │   ├── ui/
    │       └── JavaPlaygroundActivity.kt
    │   │
    │   ├── annotation/              # 自定义注解练习
    │   │   ├── GET.java
    │   │   └── Path.java
    │   ├── proxy/                   # 动态代理练习
    │   │   ├── ToyRetrofit.java     # 【亮点】手写简易版 Retrofit 核心原理
    │   │   └── InvocationHandlerImpl.java
    │   ├── generics/                # 泛型擦除与获取
    │   │   └── TypeToken.java       
    │   └── reflection/              # 反射黑科技
    │       └── Reflector.java
    │
    └── src/test/java/               # 单元测试 (面试加分项)
        └── ToyRetrofitTest.java
```

### ⚡ B. 算法与 JNI 实验室 (:labs:algo)

*考察点：NDK, C++, CMake, 性能对比*

```
labs/
└── algo/
    ├── src/main/cpp/                # Native 代码目录
    │   ├── CMakeLists.txt
    │   ├── native-lib.cpp           # JNI 接口层
    │   └── core/
    │       └── sort_algo.cpp        # 纯 C++ 算法实现
    │
    └── src/main/java/com/TechAtlas/lab/algo/
        ├── NativeLib.kt             # external fun 定义
        └── ui/
            └── AlgoComparisonScreen.kt # 展示 Java vs C++ 排序耗时对比图表
```

### 📉 C. 性能优化实验室 (:labs:performance)

*考察点：Trace, 内存泄露, 启动优化*

```
labs/
└── performance/
    ├── src/main/java/com/TechAtlas/lab/performance/
    │   ├── memory/
    │   │   └── LeakingActivity.kt   # 模拟 Handler/Static 内存泄露
    │   ├── trace/
    │   │   └── LaggyView.kt         # 模拟掉帧，结合 androidx.tracing 演示
    │   └── ui/
    │       └── PerformanceLabScreen.kt
    └── src/debug/                   # 仅在 Debug 包引入 LeakCanary
```
