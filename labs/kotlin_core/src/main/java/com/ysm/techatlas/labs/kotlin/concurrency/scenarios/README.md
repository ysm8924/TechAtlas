# Android 多线程实战场景指南

本模块旨在将抽象的并发关键字（锁、同步、异步）与 Android 真实的开发场景挂钩，帮助你建立直观的体系化认知。

## 场景 A：保护共享资源 (数据安全)
*   **场景**：App 运行过程中，多个后台任务（如埋点、配置同步、推送）同时需要读写同一个内存缓存 `Map`。
*   **工具**：
    *   `synchronized` / `ReentrantLock`：传统的 Java 阻塞锁。
    *   `Mutex`：Kotlin 协程专用非阻塞锁。
*   **面试考点**：为什么不能在 UI 线程用 `synchronized` 锁一个耗时很长的任务？（答：会阻塞 UI 线程，即便主线程不干活，它也会因为等锁而 ANR）。

## 场景 B：任务的“排队”与“合并” (逻辑顺序)
*   **场景**：
    1.  串行：必须先执行 A 接口，拿着 A 的结果去调 B。
    2.  并行合并：同时请求广告接口和配置接口，两个都回来了再关闭闪屏页。
*   **工具**：协程的 `async/await`。
*   **面试考点**：对比传统的 `Handler.sendMessage` 或接口回调，协程的优势在哪？（答：结构化并发、消灭回调地狱、提升代码可读性）。

## 场景 C：资源限流 (防止系统过载)
*   **场景**：图片加载框架（如 Glide）或者下载器，不能有多少图就开多少线程。
*   **工具**：`Semaphore` 或自定义 `Dispatcher`。
