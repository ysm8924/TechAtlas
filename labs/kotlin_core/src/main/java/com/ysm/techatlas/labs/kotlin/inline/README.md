# 内联函数 (Inline Functions) 核心考点总结

## 1. 为什么需要 inline？（基础语法糖背后的代价）
在 Kotlin 中，高阶函数（以函数作为参数或返回值的函数）非常方便，但在 JVM 层面，**每一个 Lambda 表达式都会被编译成一个匿名类（如 `Function0`, `Function1`）的实例**。
如果在循环中频繁调用高阶函数，会产生大量的对象分配，导致内存抖动（Memory Churn）和频繁的 GC。

**`inline` 的作用**：在编译阶段，将内联函数自身的代码，以及传递给它的 Lambda 表达式的代码，直接**“复制粘贴”**到调用处。
**收益**：消除了对象的创建开销和虚方法调用开销。

## 2. 核心关键字解析
*   **`inline`**：开启内联。通常只建议对**包含 Lambda 参数的高阶函数**使用。对于普通函数，JVM 原本就有自己的内联优化（JIT），手动加 `inline` 反而会导致编译产物体积无意义膨胀。
*   **`noinline`**：局部拒签。当一个内联函数有多个 Lambda 参数，但你希望把其中某个 Lambda 当作真正的对象存起来（或传递给非内联函数）时，必须用 `noinline` 标记它，使其保持对象形态。
*   **`crossinline`**：禁止非局部返回。在 `inline` 函数的 Lambda 中，默认允许使用 `return` 直接结束外层函数（这叫非局部返回）。但如果你的 Lambda 被放在了另一个线程或局部对象中执行，这种 `return` 会破坏调用栈。`crossinline` 强制要求 Lambda 内只能使用 `return@label`，保证控制流安全。
*   **`reified`**：泛型具化。Java/Kotlin 有泛型擦除机制（运行时不知道 `T` 是什么）。如果配合 `inline`，因为代码被直接复制到了调用处，编译器能在调用点知道确切的类型，从而通过 `reified` 关键字在运行时获取 `T::class.java`。

## 3. 面试防坑指南
1.  **大函数不要 inline**：会导致编译后的字节码急剧膨胀（Code Bloat）。
2.  **私有成员访问限制**：`inline` 函数如果是 `public` 的，它内部不能访问所在类的 `private` 成员（因为它的代码会被复制到别的地方，那里没有访问权限）。
